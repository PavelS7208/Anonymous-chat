## Реализация Этап 3 

### Результаты собственного ревью этапа 2

- Функционал слоев "протекает". Затруднено тестирование как обособленных структур. Необходимо частичное перепроектирование разделение на слой app-service-домен и т.п.
- Некоторые методы и объекты нарушают принцип Единой ответсвенности, много разных действий в одном, например и хранение комнат и организация стрима. Провести анализ.
- Обнаружены Go антипатерны, возвращаем из конструкторов интрфейсы, а не методы; интерфейсы иногда объявлены по месту реализации, а не использования и т.д. Проветси анализ.
- Логирование что есть включено в доменный уровень chat. Убрать на границы вызовов, а не внутри.
- Нет никакой защиты от сетевых атак. Проработать и добавить.
- Логирование по уровням отсутствует
- Ошибоки на уровне http есть, но не покрывают всех веток
- Тесты слабые, покрытие низкое. На уровне http их нет совсем и они затруднены при такой архитектуре.
- Не везде удачный нейминг типов и методов
