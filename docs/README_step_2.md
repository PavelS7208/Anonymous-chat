Добавить тест на идемпотентность Close() для Member. Убедитесь, что повторные вызовы Close() действительно безопасны.\
\
*Решено частично* Подумать, может ассинхронно сделать

### Особенности решения

- Разделение по слоям. Слой логики отделен от обработчиков в полном объеме, нет зависимостей
- Явно выделен сервисный слой (оркестрация)
- Возможность тестирования отдельно каждого элемента
- Приватный ключ Ed25519 не хранится, он используется для отправки при подключении и удаляется сразу
- Продвинутый маппинг ошибок от слоя сущностей, на бизнес и на httStatus
- Возможность задать при запуске все параметры, нет "магических" значений в коде
- Корректно работающий механизм остановки сервера (закрытие всех комнат) 



### Пример кода main

Параметры приложения задаются при запуске сервера
``` go
app := NewAnonymousChatApp(
    WithServerAddr(":8080"),
    WithTimeouts(5*time.Second, 60*time.Second),
    WithChatConfig(
        //chat.WithInitialHistoryLength(10),  // По дефолту InitialHistoryLength стоит 10 как в ТЗ
        chat.WithMaxHistoryStorageLength(1000),
        //chat.WithMaxMemberID(1000),  // По дефолту как в тз (1<<63)-1
        // chat.WithInitialHistoryCap(3000), // можно переопределить, по дефолту длина*2
        // chat.WithEventChannelBuf(64),    // По дефолту сейчас 64 (без тестов, просто на глаз)
    ),
    WithServiceConfig(
         //service.WithMaxMessageBytes(512),   // По дефолту 1024 как в ТЗ, можно поменять
         service.WithRoomNamePattern(`^[a-zA-Z0-9_-]{1,64}$`), // По дефолту длина имени 32
    ),
    // WithLogger(customLogger), // при необходимости
)

if err := app.Run(); err != nil {
    log.Printf("Server error: %v", err)
}
```
