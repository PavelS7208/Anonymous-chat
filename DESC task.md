Реализовать сервер анонимного HTTP-чата со следующим протоколом:

Участник чата подключается к произвольной комнате используя `GET /{room}` и получает 
HTTP 1.1 chunked (https://en.wikipedia.org/wiki/Chunked_transfer_encoding) ответ с потоком событий.
События терминируются символом переноса строки внутри http chunk'а.
Первое событие содержит уникальный в пределах комнаты и сессии сервера возрастающий номер участника и 
случайно сгенерированный приватный ed25519 ключ (https://en.wikipedia.org/wiki/EdDSA#Ed25519).
Последующие события включают до 10 событий истории чата данной комнаты, 
все сообщения отправленные участниками данной комнаты с момента подключения,
а так же события информирующие о подключении и отключении к комнате участников чата.

Чтобы отправить сообщение в комнату, пользователь может вызвать `POST /{room}` с телом содержащим 
публичный ключ выведенный из приватного ключа полученного при подключении, подпись и тело сообщения.

# Эндпоинты
  
## GET /{room}

Возвращает HTTP 1.1 chunked ответ с потоком событий комнаты.
Каждое событие отправляется в составе одного HTTP chunk'а.

Первое (инициализирующее) событие получаемое сразу после подключения имеет формат

```
{id} {privkey}\n
```

- `{id}` - уникальный номер участника, от 1 до 2^63-1
- `{privkey}` - base64 содержащий 32 байта приватного ed25519 ключа в формате RFC 8032 (без копии публичной части).

Все последующие события, включая события истории комнаты имеют формат

```
{timestamp} {id} {verb} {message}\n
```

- `{timestamp}` - число секунд от начала UNIX-эпохи
- `{id}` - уникальный номер участника который привёл к генерации данного события
- `{verb}` - одно из `joined`, `left` или `:`
- `{message}` - опциональное тело сообщения в UTF-8, ограничено длиной в 1024 байта,
  не может начинаться или заканчиваться пробельными символами,
  не может содержать символа переноса строки,
  не может содержать некорректных или незаконченных UTF-8 последовательностей.

После получения новым участником инициализирующего события и событий истории комнаты, участники комнаты получают событие вида

```
{timestamp} {id} joined\n
```

И это событие становится частью истории комнаты.

В момент отключения участника, все оставшиеся участники комнаты получают событие вида

```
{timestamp} {id} left\n
```

И это событие становится частью истории комнаты.

## POST /{room}

Участник может отправить сообщение в комнату используя данный эндпоинт, ожидаемый content-type: text/plain и тело в формате

```
{pubkey} {signature} {message}\n
```

- `{pubkey}` - base64 содержащий 32 байта публичного ed25519 ключа выведенного из приватного ключа в инициализирующем событии
- `{signature}` - base64 содержащий 64 байта ed25519-подписи последующего сообщения
- `{message}` - тело сообщения чата


В момент отправки участником сообщения, все участники комнаты получают событие вида

```
{timestamp} {id} : {message}\n
```

И это событие становится частью истории комнаты.

> \r\n

< Transfer-Encoding: chunked\r\n
< \r\n

< 13\r\n1773685410 2 : abc\n\r\n
< 12\r\n1773685412 2 left\n\r\n
